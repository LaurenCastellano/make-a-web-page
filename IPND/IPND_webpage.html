♦ Lesson 1 Notes ♦

What are we focusing on?

  • Procedural Thinking - Breaking it into steps. Involves creating perfectly clear and unambiguous
    instructions for a computer to follow
  • Abstract Thinking - Apples and oranges example. finding similarity, or as programmers would
    say, generality amongst seemingly different things
  • Systematic Debugging - Identify the problem, break it down to what caused it then fix it
  • Technological Empathy - Start thinking like a computer and empathize with what it will
    understand.

Basics sof Web and HTML

  • HTML - (Hyper Text Markup Language)main doc type of the web
  • URLs - how you refer to docs on the web
  • HTTP - (Hyper Text Transfer Protocol) protocol that the web runs on

Major pieces of the web

  • Browser - a program that runs on your computer to display files found on the web
  • Internet - worlds largest computer network
  • HTTP - the main protocol of the web
  • Servers - Computers that host the files that make up the web

HTML

  • Heart of the web. The text content is what you see and markup is what it looks like. It references
    other docs such as images or videos and can link to other pages.

  • HTML Markup

    • Tags
      • <name> Content </name>
      • The closing tag has a / in it.
      • The whole this is an element
      • Void tags don't have a closing tag
      • Attributes - are a property of an HTML element like the herf for a link and the anchor
        is the <a>

    • Images
      • <img>
      • <img src="url" alt="add discriptive text for the blind">
      • Doesn't need closing tag because it is a void'

    • Inline vs. Block
      • Inline happens within a line of text but a block creates an invisible box around your text

    High level HTML set up

    • <!Doctype HTML> - This tells the doc that it is an HTML doc...
    • <html> opening and closing the whole document
    • <head> - Meta data if you include Java or Css it would go here
      • <title> </title> - the title of the page
    • <body> </body> - The body of the page
    • </html> - close it out

    ♦ Lesson 2 Notes ♦

    Website is like a house

    • HTML is the structure where are the wells
      • HTML controls the "structure" of a web page
    • CSS is the style of the house what color are the walls
    • JAVA script is the interactive component - garage door opener, tv remote. Things that change things

    • HTML HyperText Markup Language is the standard markup language used to create web pages.
    • CSS Cascading Style Sheets this is a style sheet language used for describing the look and formatting of
      a document written in a markup language.
    • DOM Document Object Model a cross-platform and language-independent convention for representing and
      interacting with objects in HTML (and other markup languages). The nodes of every document are organized
      in a tree structure, called the DOM tree.
      • When programmers talk about the "DOM" they are talking about the tree-like structure of a page.

    ◊ Running list of tags ◊

    • Bold - <b> </b>
    • Italics - <em> </em>
    • Line Breal - <br> </br>
    • Paragraph - <p> </p>
    • Link - <a href= website>what you want it to say </a>
    • SPAN - <span> </span>
      • This creates an inline small section of HTML. It creates the chunk and you
        can hook some information onto that chunk.
    • DIV - <div> </div>
      • This creates a block-line element which is basically equivalent to having a
        line-break before and after it and it is used to group larger chunks of code.
    • Link to Style sheet - <link rel="style" type="text/css" href="name of doc">
